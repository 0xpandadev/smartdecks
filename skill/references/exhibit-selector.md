# Exhibit Selector

Choose the exhibit before writing detailed copy. A slide without an exhibit choice tends to become a box layout.

| Content Need | Preferred Exhibit |
| --- | --- |
| Trend over time | Line chart |
| Category comparison | Bar chart |
| Composition | Stacked bar or 100% stacked bar |
| Change bridge | Waterfall |
| Funnel/conversion | Funnel or staged bar |
| Option evaluation | Scorecard |
| Trade-off/prioritization | 2x2 matrix |
| Process | Horizontal flow |
| Timeline | Roadmap |
| System or operating model | Layered architecture |
| Causal structure | Logic diagram |
| Problem decomposition | Issue tree |
| Before/after state | Split comparison |
| Portfolio | Bubble chart |
| Capability maturity | Maturity curve |
| Summary | 3-card or 4-card message layout |
| Named companies/tools | Logo/icon comparison grid |
| Abstract capability | Icon-led capability map |
| Argument with evidence | Claim-evidence-implication layout |
| Jobs/tasks decomposition | Task stack or unbundling map |
| AI suitability | Heatmap or scored bar chart |
| Role change | Before/after role architecture |

## Consulting Exhibit Rules

- Put the conclusion in the action title.
- Label the chart directly where possible.
- Use callouts to explain the implication, not the obvious.
- Use source and note lines on data slides.
- Avoid chart junk, 3D effects, and unneeded icons.
- Tables should have a clear reading path and highlighted conclusion.

## Minimum Visual Standard

For each slide, pick one primary visual type:
- `chart`
- `diagram`
- `table`
- `matrix`
- `timeline`
- `icon_grid`
- `logo_map`
- `hero_message`
- `text_summary`

Use `text_summary` only for executive summary, section transition, or closing slides. If more than 30% of the deck uses `text_summary` or generic cards, redesign the deck with charts or diagrams.

## MBB-Level Diagram Standard

Good consulting diagrams do three jobs:
- Structure the argument
- Show direction or causality
- Make the "so what" visually obvious

Avoid diagrams that are merely decorative.人物、ロゴ、アイコンは意味を持つラベルや矢印と一緒に使う。
