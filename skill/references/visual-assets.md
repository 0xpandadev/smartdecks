# Icons, Logos, and Visual Assets

Use icons and logos when they reduce cognitive load or identify real entities. Do not sprinkle icons as decoration.

## Icon Rules

Use icons for:
- Capability categories
- Workflow steps
- Roles
- Risks
- Enablers
- Technology layers
- Decision points

Rules:
- Use a consistent icon family within a deck.
- Prefer simple line icons for consulting and executive decks.
- Use filled or bold icons only for expressive design systems.
- Keep icon stroke weight consistent with the template.
- Pair icons with labels; unlabeled icons are usually too vague.

## Logo Rules

Use logos for:
- Company comparisons
- Ecosystem maps
- Tool stacks
- Source attribution when appropriate

Rules:
- Use official logos only when available and appropriate.
- Do not distort logos.
- Do not imply endorsement.
- If logo rights or assets are uncertain, use a neutral wordmark tile instead.
- For AI/tool ecosystems, a clean logo grid or stack map is often better than prose.

## Visual Asset Selection

When building from an article:
- Extract named organizations, products, frameworks, and source links.
- Decide whether each should become a logo tile, source footer, or plain text reference.
- Use abstract icons for generic roles such as legal, accounting, marketing, PM, governance, and risk.

## Consulting Visual Vocabulary

Use:
- Thin arrows for flow
- Highlight bands for implication
- Numbered callouts for reasoning
- Small multiples for repeated comparisons
- Swimlanes for ownership
- Heatmaps for qualitative scoring
- Logo tiles for ecosystem/tool comparison

Avoid:
- Generic card grids on every slide
- Decorative emoji
- Overly cute illustrations for executive decks
- Unexplained icons
