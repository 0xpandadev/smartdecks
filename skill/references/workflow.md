# SmartDecks Workflow

## Inputs

Accept rough notes, Markdown, PDFs, research summaries, meeting notes, CSV/Excel-derived tables, URLs, images, existing PPTX files, brand guidelines, and slide templates.

If details are missing, infer reasonable defaults and proceed. Ask only when the missing choice materially changes the deck.

## Brief Fields

Capture:
- Objective
- Audience
- Occasion
- Output format
- Desired slide count
- Tone
- Main message
- Evidence and sources
- Brand/template constraints
- Must-include topics
- Must-avoid claims

## Operating Modes

Fast mode:
- Use when the request is clear.
- Build and render without asking for approval on the slide plan.

Review mode:
- Use for high-stakes, long, client-facing, investor, board, or ambiguous decks.
- Show the storyline and slide list before rendering.

Schema mode:
- Use when the user asks for a reusable structure, data contract, or automation pipeline.

## Deliverable Contract

When making a deck, finish with:
- Created/updated file path(s)
- Slide count
- Template used
- Assumptions worth knowing
- QA status

Do not over-explain the internal process unless the user asks.
