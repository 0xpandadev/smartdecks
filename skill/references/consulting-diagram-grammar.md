# Consulting Diagram Grammar

Choose the diagram before writing slide body text. The diagram should carry the argument.

## Pattern Selector

| Information Need | Diagram Pattern | Use When |
| --- | --- | --- |
| Decompose a problem | Issue tree | The user provides many issues or drivers |
| Show how value is created | Value chain | Inputs transform into outputs or impact |
| Explain a system | Operating model | People, process, data, tools, governance interact |
| Show responsibility | Swimlane | Multiple actors own different steps |
| Compare options | Scorecard | There are criteria and alternatives |
| Prioritize | 2x2 matrix | There are two meaningful axes |
| Explain cause/effect | Causal loop | Feedback loops or reinforcing dynamics matter |
| Show transition | Before/after architecture | The point is a structural shift |
| Show maturity | Maturity curve | Progression from low to high capability matters |
| Show decision logic | Decision tree | The audience must choose based on conditions |
| Show portfolio exposure | Bubble map | Size, risk, and category matter together |
| Show evidence trail | Claim-evidence-implication | The slide must prove a point |

## Diagram Requirements

Every diagram must have:

- A title that states the conclusion.
- Labeled elements, not anonymous shapes.
- Directionality: arrows, hierarchy, sequence, axes, or containment.
- A highlighted implication.
- No decorative-only objects.

## Layout Moves

Use these instead of generic boxes:

- Left-to-right transformation: input -> analysis -> decision.
- Top-down logic: thesis -> drivers -> evidence.
- Center-out system: core mechanism with surrounding actors.
- Two-lane split: AI-owned work vs human-owned decisions.
- Evidence stack: source -> observation -> implication.
- Heatmap: categories by intensity or suitability.
- Waterfall/bridge: baseline -> drivers -> outcome.

## Five-Second Rule

If the reader cannot understand the main structure in five seconds, simplify:

- Reduce nodes.
- Merge categories.
- Add direct labels.
- Highlight one path.
- Move caveats to footnotes or appendix.
