# Consulting Diagrams

Use diagrams to explain structure, causality, and decisions. Do not use decorative illustrations that merely fill space.

## Default Diagram Prompt

Use this as the base visual style for generated or described illustrations:

```text
Minimalist flat line art illustration, a simple character, clean thin black outlines, limited color palette, splashes of neon pistachio green as accent color, professional vector style, white background, friendly and modern aesthetic, high-end tech corporate style.
```

Add the business logic:

```text
Make it suitable for an executive consulting deck. Show labeled business logic, arrows, causal relationships, and a clear one-slide message. Use the illustration to explain the process, system, decision, or value creation mechanism, not as decoration.
```

## Diagram Types

- Value creation loop
- AI workflow
- Human-in-the-loop decision process
- Capability factory
- Before/after operating model
- Three-step transformation
- Data-to-action pipeline
- Skill supply chain
- Issue tree
- Governance model
- Flywheel
- Layered architecture

## Rules

- Diagram title must match the slide action title.
- Use labels on every important node.
- Show directionality with arrows.
- Keep one main flow per slide.
- Use green only for emphasis, current focus, or target state.
- Use people sparingly; they should represent roles, not decoration.
- Prefer simple rectangles, lines, arrows, and callouts for PPTX compatibility.
- Add icons or logo tiles when they identify real tools, companies, or roles.
- Add a callout band or "so what" at the bottom when the diagram is complex.

## Diagram Quality Bar

Before rendering, ask:
- What claim does this diagram prove?
- Where does the reader's eye start?
- Where does it end?
- Which element is highlighted as the answer?
- Can one label be removed without losing meaning?

If the answers are unclear, simplify the diagram.

## Common Pattern: AI Skill Factory

Structure:
1. Raw work demand
2. Skill design
3. Agent/tool packaging
4. Deployment into workflows
5. Usage data and feedback
6. Continuous improvement

Recommended exhibit:
- Circular flywheel or left-to-right factory line with feedback loop.

## Common Pattern: Zeroth Economy

Structure:
1. Human intent
2. AI agents
3. Automated transactions/tasks
4. New coordination layer
5. Economic outcomes

Recommended exhibit:
- Layered system map with human layer, agent layer, transaction layer, and market/outcome layer.
