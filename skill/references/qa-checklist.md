# QA Checklist

Run before final delivery.

## Story QA

- Does the deck answer the user's objective?
- Is the audience clear?
- Does each section advance the argument?
- Is there a clear recommendation or next step when appropriate?
- Are assumptions separated from facts?

## Slide QA

- One slide, one message
- Action title states a conclusion
- Main exhibit supports the title
- No dense paragraph walls
- Appendix used for details
- Sources included where needed

## Visual QA

- No text overflow
- No object overlap
- No slide requires scrolling in the intended viewport
- Text stays inside the slide safe area
- Consistent margins
- Consistent fonts and sizes
- Clear chart labels
- Readable tables
- Sufficient contrast
- Page numbers and footers consistent
- No accidental template mixing
- Deck does not overuse generic cards/boxes
- Charts, diagrams, icons, logos, and tables are used where they clarify the message

## Data and Source QA

- Sources are named clearly
- Dates are included when relevant
- Calculations are noted if not obvious
- Unsupported claims are softened or removed

## Final Response

Report:
- File path(s)
- Slide count
- Template
- QA status
- Any unresolved issues
