# Semantic Icons

Icons are allowed and encouraged when they replace words or make structure easier to scan. Icons are forbidden when they are decoration.

## Icon Meaning Test

Use an icon only if:

- Removing it would make the slide slower to understand.
- It represents a repeated concept across the deck.
- It labels a role, entity, risk, capability, data source, workflow step, or decision point.

Pair icons with short labels unless the icon is universally obvious in context.

## Core Icon Vocabulary

| Concept | Use For |
| --- | --- |
| Data | datasets, evidence, logs, filings, metrics |
| AI | model, agent, automation, inference |
| Human | judgment, accountability, expert review |
| Risk | uncertainty, exposure, failure condition |
| Governance | policy, approval, controls, compliance |
| Cashflow | money, budget, liquidity, investment capacity |
| Decision | go/no-go, trade-off, branching logic |
| Workflow | process steps, operating rhythm |
| Source | citation, document, report, transcript |
| Feedback | learning loop, iteration, monitoring |
| Customer | user, segment, audience, stakeholder |
| Tool | platform, connector, application layer |

## Placement Patterns

- Use 3-5 icons across the top for category labels.
- Use icons at swimlane row headers.
- Use icons inside process nodes when the node repeats across slides.
- Use icons in comparison tables only when they speed scanning.
- Use logo or wordmark tiles for named companies/tools instead of generic icons.

## Prohibited

- Icons in corners just to fill whitespace.
- Multiple icon styles in one deck.
- Decorative pictograms that duplicate nearby text without adding scan value.
- Icons without labels in complex or executive slides.
