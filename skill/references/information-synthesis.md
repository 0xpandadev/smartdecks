# Information Synthesis

SmartDecks should not turn input into a prettier summary. It should convert raw material into a meaningful deck argument.

## Synthesis Ladder

For each source, move up this ladder before designing slides:

1. Extract facts: events, numbers, actors, dates, claims, sources.
2. Identify the central tension: what is changing, contested, risky, or misunderstood.
3. Separate evidence from interpretation.
4. Find causal links: because X changed, Y becomes possible or risky.
5. Identify audience implications: what the reader should believe, decide, monitor, or do.
6. Convert the implication into a slide message and visual exhibit.

Do not stop at level 1 or 2 unless the user explicitly asks for a summary.

## Slide Meaning Test

Every slide must answer:

- What is the non-obvious point?
- What evidence supports it?
- What structure makes it easier to understand?
- What should the reader take away in five seconds?
- What action, decision, or monitoring question follows?

If a slide only restates text from the input, redesign it.

## Compression Patterns

Use these transformations:

| Raw Input Pattern | Deck Transformation |
| --- | --- |
| Long explanation | Claim -> evidence -> implication |
| Many examples | Group into 3-5 categories |
| Process description | Flow, swimlane, or operating model |
| Trade-off | 2x2 matrix or scorecard |
| Stakeholder complexity | Stakeholder map or ecosystem map |
| Risk discussion | Risk heatmap or mitigation table |
| Before/after change | Before/after architecture |
| Many tools/entities | Logo map or capability stack |
| Abstract thesis | Causal loop or value chain |

## Insight Quality Bar

A good slide insight:

- Is more specific than the article heading.
- Names a mechanism, trade-off, risk, or decision.
- Can be visualized without relying on paragraphs.
- Has a clear so-what.

Weak:
- "AI is changing finance."

Stronger:
- "AI finance shifts value from stock prediction to repeatable evidence workflows."

## Prohibited Behavior

- Do not merely shorten paragraphs.
- Do not create one card for every paragraph.
- Do not use icons as decoration.
- Do not make a slide if it has no tension, decision, mechanism, or evidence.
