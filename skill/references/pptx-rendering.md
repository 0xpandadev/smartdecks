# PPTX Rendering

Use PPTX when the user asks for PowerPoint, corporate slides, board materials, editable deliverables, or existing deck edits.

## Preferred Process

1. Inspect any provided PPTX/template first.
2. Create or update the `slide_plan`.
3. Map each slide to a layout.
4. Render with the available PPTX creation/editing tools.
5. Export or render screenshots when possible.
6. Fix text overflow, object overlap, missing sources, and inconsistent styles.

## Template Handling

If the user provides a corporate template:
- Preserve master layouts where practical.
- Reuse theme fonts, colors, footer, and page numbers.
- Do not introduce unrelated decorative systems.

If no template is provided:
- Use `Executive Analytical Minimalism` for business decks.
- Use 16:9 widescreen.
- Use consistent header/footer regions.
- Use a restrained palette.

## PPTX Layout Rules

- Leave clear margins.
- Use action title at top.
- Put the main exhibit in the center.
- Put sources in the lower-left or footer.
- Put page number in the lower-right.
- Use editable shapes/charts where practical.
- Avoid rasterizing all content unless visual fidelity is more important than editability.

## QA

Always inspect final slides visually when possible. Pay special attention to Japanese text wrapping, line height, table density, and source footers.
